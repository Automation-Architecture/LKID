04

Integration contract

Suggested endpoints — rename freely, but the shape should be preserved so the pages wire cleanly.

POST /api/capture

Called when the user submits the Email Gate. Stores name + email + pending lab payload. Returns a session token used by subsequent calls.

Request: { name, email, labs: { age, sex, creatinine, bun, hemoglobin?, glucose? } }

Response: { token }

POST /api/predict

Consumes the stored labs, returns the 4 kidney‑function scenarios. The Results page plots series directly into its SVG.

Request: { token }

Response:

  • series: [{ id: 'healthy' | 'stable' | 'higher' | 'no_treatment', points: [{ month, gfr }] }]
  • scenarios: [{ id, bun_target, end_creatinine, risk_band }]
  • patient: { name, age, sex }

GET /api/pdf?token=…

Server renders PDF Report.html via Chromium headless and returns application/pdf.

The same endpoint is called by the worker that sends the email, then the PDF is attached to the message generated from Email Template.html.

05

Accessibility & QA

Must‑haves

  • All interactive targets ≥ 44×44px (enforced on mobile nav, form inputs, and CTAs).
  • Colour contrast ≥ WCAG AA — navy on white, white on navy, and body‑grey on white all pass.
  • Every form field has a visible label above the input (not placeholder‑only).
  • Focus ring: 3px rgba(31,37,119,.12) + 1px --navy border.
  • Modal (Email Gate) traps focus and is closeable via Esc.

QA scenarios

  • Submit Lab Form with 0 values → four inline errors.
  • Toggle optional fields, fill one, toggle off → values persist on re‑open.
  • Submit Email Gate with invalid email → inline error, button stays enabled.
  • Refresh Results page → session restored from token, not re‑prompted for email.
  • Print Results → only the PDF layout shows; nav, buttons hidden via @media print.
